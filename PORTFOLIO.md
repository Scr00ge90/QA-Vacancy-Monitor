## Для портфолио (развёрнутое описание)

### Что это и зачем

По сути — **универсальный инструмент мониторинга Telegram-каналов** по ключевым словам с автоматической реакцией. В моём случае применяется для поиска вакансий QA, но архитектура полностью не привязана к этому сценарию.

**Логика работы проста и универсальна:**
1. Подписываешься на любые публичные каналы
2. Задаёшь ключевые слова (что искать) и слова-исключения (что игнорировать)
3. При совпадении — скрипт извлекает контакт из сообщения и отправляет заданный текст

**Другие возможные применения того же инструмента:**
- Мониторинг аренды/продажи недвижимости (ключевые слова: "сдам", "1-комн"; исключения: "агентство", "комиссия")
- Отслеживание скидок и акций ("скидка", "распродажа", "-50%"; исключить: "реклама")
- Поиск фриланс-заказов по специальности
- Мониторинг новостей по теме (крипта, акции, события)
- Любой сценарий: найди сообщение с X, без Y, сделай Z

**Гибкость обеспечивается:**
- Полностью настраиваемые списки каналов, ключевых слов и исключений через UI
- Шаблон сообщения-отклика настраивается под любой сценарий
- Прикреплённый файл (резюме, портфолио, КП) — опционально
- Менять код не нужно — всё через веб-интерфейс

Для HH.ru добавлен отдельный модуль на Selenium с собственными фильтрами: регион, опыт, зарплата, график работы, период публикации.

Проект показывает не только умение писать код, но и QA-мышление: я тестировал собственную систему, находил баги, документировал их и чинил.

### Технические решения

**Telegram интеграция через MTProto API (Telethon)**
Не Bot API, а полноценный клиент — мониторинг публичных каналов в реальном времени через `events.NewMessage`. Это сложнее чем обычные боты и демонстрирует понимание Telegram-протокола.

**Асинхронность (asyncio)**
Скрипт обрабатывает события не блокируя основной поток. При FloodWaitError читает время ожидания из исключения и делает точную паузу вместо произвольной.

**Selenium для HH.ru**
HH закрыл API для соискателей в 2025 году. Решение — браузерная автоматизация. Скрипт открывает Chrome, авторизуется (один раз, потом cookies), ищет вакансии, кликает "Откликнуться", вставляет сопроводительное письмо. Именно этим занимаются QA-автоматизаторы.

**Модульная архитектура FastAPI**
Бэкенд разбит на роутеры по ответственности: `tg_routes`, `hh_routes`, `auth_routes`, `config_routes`. Каждый модуль независим — стандартная структура production-проекта.

**Конфигурация через UI**
Все параметры меняются через веб-интерфейс без редактирования кода. Секреты (API ключи) хранятся только в `.env`, не попадают в `config.json` и не коммитятся в git.

### QA-практики применённые в проекте

**Тест-кейсы:**
- SAFE_MODE как тестовый стенд — скрипт логирует найденные контакты без реальной отправки
- 8 задокументированных тест-кейсов на основную логику фильтрации и лимитов
- Граничные условия: что будет при достижении лимита в середине обработки сообщения

**Найденные и исправленные баги:**
- Race condition при multiple usernames в одном сообщении (лимит превышался)
- Memory leak: конфиг читался с диска при каждом входящем сообщении
- Security: API hash сохранялся в открытом виде в config.json
- Сессионный конфликт: два Telethon клиента на одном файле сессии
- PID-файл не очищался после краша процесса

**Рефакторинг:**
- Выявил дублирование кода (PID логика × 2, одинаковые фильтры)
- Разделил монолитный `app.py` (~350 строк) на 4 модуля
- Разделил `index.html` (~1200 строк) на HTML + CSS + JS

## Ответы на вопросы рекрутера

**"Почему Selenium, а не официальный API HH?"**
HH.ru прекратил поддержку API для соискателей в 2025 году. Selenium — единственный способ автоматизировать отклики. Это стандартная задача для QA-автоматизатора.

**"Насколько это законно?"**
Инструмент автоматизирует действия которые пользователь и так совершает вручную. Используется в личных целях для поиска работы. Аналогично любому другому автокликеру.

**"Какие были сложности?"**
Главная — конкурентный доступ к Telegram-сессии из двух процессов одновременно. Решил разделением на `session` (монитор) и `session_web` (чтение чатов в UI). Также потратил время на правильную обработку FloodWaitError — нужно читать конкретное время ожидания из объекта исключения, а не ставить произвольную паузу.
